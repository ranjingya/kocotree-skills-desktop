# Kocotree Skills 术语表

| 术语 | 定义 |
| --- | --- |
| Skill | 由 `SKILL.md` 及可选脚本、参考资料等文件组成的能力包。 |
| Skill 聚合 | 平台中由稳定 `skillId` 标识、包含多个不可变版本的 Skill 记录。 |
| Skill 名称（`skillName`） | `SKILL.md` frontmatter 中的 `name`。平台按不区分大小写的方式保证全局唯一，创建后不可修改，用于本地目录和手动调用。 |
| 展示名称（`displayName`） | 平台界面使用的名称，由当前 Owner 维护；允许重复，但创建或修改时需要确认重名风险。 |
| Skill 描述（`skillDescription`） | `SKILL.md` frontmatter 中的 `description`，随具体版本保存，并参与 Agent 自动触发判断。 |
| 展示简介（`displayDescription`） | 平台单独保存的简介，由 Owner 或协作者维护，不写回 ZIP。 |
| Owner | Skill 当前所有者。首个版本上传者成为初始 Owner；所有权可以按规则转移。 |
| 协作者 | 至少成功发布过一个版本的用户。协作者身份是永久贡献记录，不因版本撤回或所有权变化而删除。 |
| 版本上传者（`uploadedBy`） | 实际发布某个 `SkillVersion` 的用户。原始创建者可从首个版本的上传者得到，不单独保存 `createdBy`。 |
| 最近更新者（`updatedBy`） | 最近发布版本或修改平台展示信息的用户。 |
| 历任 Owner | 从所有权转移审计记录得到的历史所有者；未发布版本的前 Owner 不因此成为协作者。 |
| 平台管理员 | 由服务端配置的飞书账号列表确定、用于处理停用 Owner 和平台治理的全局角色。 |
| 部门路径（`departmentPath`） | 飞书提供的从上级部门到用户直接所属部门的名称数组，例如 `["运营中心", "运营办"]`。 |
| Skill 版本 | 由 SemVer、原始 ZIP、发布者、更新说明、状态和哈希组成的不可变记录。 |
| 当前版本 | 版本号最高且状态为 `PUBLISHED` 的版本；撤回最新版后自动回退到下一个可用版本。 |
| 撤回版本 | 状态为 `WITHDRAWN`、不能下载或重新安装但仍保留审计记录的版本。首个 `1.0.0` 不允许撤回。 |
| 派生 Skill | 从某个历史版本开始独立发展的新 Skill，拥有新的 `skillName`、Owner 和从 `1.0.0` 开始的版本线。 |
| 直接来源 | 派生 Skill 保存的 `forkedFromSkillId` 与 `forkedFromVersionId`。多级派生通过直接来源形成完整链路。 |
| 平台状态 | Skill 的在线生命周期状态，包括 `ACTIVE`、`ARCHIVED` 和 `NAME_CONFLICT`。 |
| 名称冲突（`NAME_CONFLICT`） | 已发布 Skill 的名称后来成为 Claude、Codex 或平台保留名称时进入的状态；从公开列表、搜索和公开详情隐藏，禁止安装和更新，本地已安装内容仍可使用。 |
| 归档 | Owner 或管理员填写原因后将 Skill 从公开列表、搜索和详情中移除；服务端保留完整数据。 |
| 归档凭证 | 已安装客户端按本地 `skillId` 查询到的最小归档状态，只包含状态、时间和原因。 |
| 本地解析状态 | 客户端解析 ZIP 后保存在当前上传页面内存中的预览状态，没有服务端编号和过期时间。 |
| 包哈希（`packageSha256`） | 原始 ZIP 字节的 SHA-256，用于验证下载文件未损坏或被替换。 |
| 内容哈希（`contentHash`） | 对规范化文件路径和各文件内容哈希再次计算得到的目录指纹，用于比较解压后的实际内容。 |
| 文件清单 | 某个版本去除单层外包装目录后的路径集合，记录条目类型、大小、哈希和可预览状态。 |
| 文件树 | 客户端根据文件清单构建的层级视图；文本文件可以按需预览，二进制文件只展示元信息。 |
| 安装凭证 | 客户端保存在 `~/.agents/.kocotree/` 的平台 Skill、版本、本地路径和内容哈希记录。 |
| 平台已安装 | 本地目录内容哈希与安装凭证中的平台版本一致。 |
| 平台匹配 | 没有安装凭证，但 `skillName + contentHash` 与平台历史版本完全一致，可恢复凭证但不新增安装次数。 |
| 本地已修改 | 存在安装凭证，但实际目录内容哈希与凭证记录不一致。 |
| 来源未知 | 本地存在 Skill，但没有凭证，也无法匹配平台任一历史版本。 |
| 强制安装 | 本地存在同名不同内容时，用户确认后先备份原目录，再安装平台版本的流程。 |
| 一键替换 | 安装用途相近的派生 Skill 时，先安装并验证新 Skill，再备份并移除旧 Skill 的主动切换流程。 |
| 安装事件 | 客户端完成本地安装后提交的幂等记录，用于统计成功安装操作次数。 |
| 弱提醒 | 只在账户区域显示小红点并收进通知列表的提醒方式，不弹窗、不发送系统或飞书通知。 |
| 设备授权 | 桌面客户端打开浏览器完成飞书认证，再轮询领取访问令牌的流程。 |
| 模拟接口 | 实现与真实 HTTP 客户端相同 TypeScript 接口、但在前端内返回模拟数据的实现。 |
| 服务端 DTO | API 返回的用户、Skill、版本和 Tag 数据，不包含本地安装状态。 |
| 本地安装状态 | 只存在于客户端的安装版本、修改状态、备份和 Claude 链接状态。 |
| 通用 Skill 目录 | `~/.agents/skills`，Kocotree Skills 的统一安装目标。 |
| Claude 链接 | `~/.claude/skills` 指向通用 Skill 目录的目录链接。 |
