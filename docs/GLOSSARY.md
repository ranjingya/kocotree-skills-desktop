# Kocotree Skills 术语表

| 术语 | 定义 |
| --- | --- |
| Skill | 由 `SKILL.md` 及可选脚本、参考资料等文件组成的能力包。 |
| Skill 聚合 | 平台中由一个稳定 `skillId` 标识、包含多个不可变版本的 Skill 记录。 |
| 内部名称（`internalName`） | `SKILL.md` frontmatter 中的 `name`。平台全局唯一，创建后不可修改，用于版本归属和本地目录名。 |
| 展示名称（`displayName`） | 平台界面使用的名称，可编辑、可重复，不写回 `SKILL.md`。 |
| 原始描述（`sourceDescription`） | `SKILL.md` frontmatter 中的 `description`，随版本包保存。 |
| 平台简介（`summary`） | 平台单独保存的简短说明，可编辑，不修改原始 ZIP。 |
| Slug | 适合链接和展示的可读标识，不承担 Skill 的稳定身份。 |
| Tag | 用于分类和筛选的标签。单个 Skill 最多 5 个，按规范名去重。 |
| Skill 版本 | 由 SemVer、原始 ZIP、发布者、更新说明和哈希组成的不可变记录。 |
| 最新版本 | 当前 Skill 中 SemVer 最高的已发布版本。安装历史版本不会改变它。 |
| 上传会话 | ZIP 解析后生成的临时状态，通过 `uploadId` 与发布请求关联。 |
| 包哈希（`packageSha256`） | 原始 ZIP 字节的 SHA-256，用于验证下载文件未损坏或被替换。 |
| 内容哈希（`contentHash`） | 对规范化文件路径和各文件内容哈希再次计算得到的目录指纹，用于比较解压后的内容。 |
| 安装凭证 | 客户端记录平台 Skill、版本、本地路径和内容哈希的本地数据。 |
| 平台已安装 | 本地目录内容哈希与安装凭证中的平台版本一致。 |
| 本地已修改 | 存在安装凭证，但实际目录内容哈希与已安装版本不一致。 |
| 来源未知 | 本地存在 Skill，但没有安装凭证，也无法匹配平台任一历史版本。 |
| 安装事件 | 客户端完成本地安装后向平台提交的幂等记录，用于统计成功安装次数。 |
| 设备授权 | 桌面客户端打开浏览器完成飞书认证，再轮询领取访问令牌的流程。 |
| 模拟接口 | 实现与真实 HTTP 客户端相同 TypeScript 接口、但在前端内返回模拟数据的实现。 |
| 服务端 DTO | 由 API 返回的用户、Skill、版本和 Tag 数据，不包含本地安装状态。 |
| 本地安装状态 | 只存在于客户端的安装版本、修改状态、备份和 Claude 链接状态。 |
| 通用 Skill 目录 | `~/.agents/skills`，Kocotree Skills 的统一安装目标。 |
| Claude 链接 | `~/.claude/skills` 指向通用 Skill 目录的目录链接。 |
