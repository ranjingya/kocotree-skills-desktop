# ADR-0003：本地安装状态与跨平台边界

- 状态：已接受
- 日期：2026-07-15

## 背景

服务端只能知道平台版本，无法可靠判断某台电脑的安装目录、手工修改、备份和 Claude 链接状态。单纯把安装状态放入服务端 DTO 会导致不同设备互相覆盖；只把凭证放在应用数据目录，又会在清理或重装应用后丢失来源。

## 决策

- 服务端 Skill DTO 不包含本地安装字段。
- 客户端统一安装到 `~/.agents/skills`。
- 安装凭证和备份保存在 `~/.agents/.kocotree/`，不写入具体 Skill。
- 平台版本保存 `packageSha256` 和规范化目录 `contentHash`。
- 客户端通过内容哈希检测本地修改，并在凭证丢失时恢复版本关联。
- 替换目录前备份，安装失败自动恢复，每个 Skill 保留最近 3 份。
- 首版实现 Windows 安装器，统一接口为未来 macOS 适配器保留边界。
- Claude 链接只在目标不存在或为空时创建；非空目录绝不自动覆盖。

## 结果

### 正面影响

- 多台设备可以拥有独立安装状态。
- 应用重装后仍可恢复平台来源。
- 本地修改不会被静默覆盖。
- 平台差异集中在安装器，不污染 React 页面。

### 代价

- 启动扫描和目录哈希会产生额外磁盘读取。
- 用户手工修改后只能判断内容不同，无法自动合并。
- Windows 与 macOS 仍需分别实现链接和权限处理。

## 后续约束

- 内容哈希算法必须在平台和客户端保持字节级一致。
- 安装事件只能在原子替换和最终校验成功后上报。
- 本地凭证写入需要使用临时文件和原子重命名，避免进程中断造成损坏。
- 删除、移动或覆盖用户目录前必须有明确确认和可恢复路径。
