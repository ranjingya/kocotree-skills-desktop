# ADR-0003：本地安装安全与跨平台边界

- 状态：已接受
- 日期：2026-07-15

## 背景

平台版本由服务端管理，实际安装目录只能由当前设备写入和校验。安装流程需要隔离在线业务与本地文件系统操作，并在同名目录冲突、进程中断和网络失败时保护用户已有内容。

## 决策

- 服务端 Skill DTO 不包含本地安装字段。
- 客户端统一安装到 `~/.agents/skills`。
- 安装凭证和备份保存在 `~/.agents/.kocotree/`，不写入具体 Skill。
- 平台版本保存 `packageSha256` 和规范化目录 `contentHash`。
- 安装器通过内容哈希判断目标目录是否与待安装版本一致。
- 目标目录内容与平台版本一致时返回已安装结果，不重复写入或增加安装次数。
- 替换目录前备份，安装失败自动恢复，每个 Skill 保留最近 3 份。
- 同名不同内容允许用户明确选择“备份并强制安装”。
- 历史版本降级和撤回版本切换都由用户确认，不自动修改本地目录。
- Windows 与 macOS 安装器使用统一接口，平台路径与权限差异由 Rust 适配器处理。
- 本地扫描、移除、手动恢复、来源识别和 Claude 链接管理不属于当前范围。

## 结果

### 正面影响

- 多台设备可以拥有独立安装状态。
- 本地修改不会被静默覆盖。
- 平台差异集中在安装器，不污染 React 页面。

### 代价

- 安装前的目标校验和目录哈希会产生额外磁盘读取。
- 同名目录内容不同只能提示冲突，无法自动合并。
- Windows 与 macOS 仍需分别实现路径、文件占用和权限处理。

## 后续约束

- 内容哈希算法必须在平台和客户端保持字节级一致。
- 安装事件只能在原子替换和最终校验成功后上报。
- 本地凭证写入需要使用临时文件和原子重命名，避免进程中断造成损坏。
- 移动或覆盖用户目录前必须有明确确认和可恢复路径。
